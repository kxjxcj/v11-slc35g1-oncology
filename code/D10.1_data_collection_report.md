### 2.3 Mouse mito gene 警告

Mouse gene naming convention 是 `mt-` (lowercase) 而非 `MT-` (uppercase)。QC 脚本中 `adata.var_names.str.startswith('MT-')` 返回 0。需要修正为 case-insensitive (e.g., `str.lower().startswith('mt-')`)。**下游 D10.2 必须修正此 bug**。

### HM-56: scRNA-seq 数据采集的范围与方法限制

| 限制 | 影响 | 缓解 |
|---|---|---|
| **L1**: 仅 2/5 用户指定数据集完整下载 | 27 基因扰动预测只能基于 2 数据集 (Dixit + Norman) | Norman 补完后 = 3/5; D10.2 可用 |

| **L3**: sc-pert figshare 4/7 链接失效 | 无法快速补全 Frangieh/Srivatsan | 直接 GEO SRA (重 ~5-10GB each) |
| **L4**: GEO accession 错误 (GSE161462 ≠ Replogle 2022) | 需重新确认 Replogle 2022 真实 accession (查文献后应为 GSE146035 K562 + GSE146036 RPE1) | 下游 worker 验证 |
| **L5**: scrublet 在大 csr_matrix 上失败 | 1 dataset 缺 doublet 过滤 | 用 Scanpy 替代 (sc.pp.scrublet via scanpy.external) |
| **L6**: 大文件下载速度 ~1-1.5 MB/s (figshare S3 限流) | 100GB 实际下载需 18-30 小时 | 接受 metadata-only 折衷 |

### HM-56 also covers data scope

- **scRNA-seq ≠ bulk**: 单细胞 UMI 计数与 v10 final 的 bulk RNA-seq (TCGA) 完全不同
- **K562/RPE1/BMDC ≠ CRC**: scRNA-seq 训练的 scGen 模型 transfer 到 CRC 病人是 out-of-distribution
- **CRISPR KO ≠ disease mutation**: Perturb-seq 是 systematic loss-of-function, 27 目标基因的 CRC 变异 (mostly missense/SNP) ≠ KO
- **0 湿实验 compatible**: 100% 公开数据 ✓
