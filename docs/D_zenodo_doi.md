## 4. Zenodo API upload (curl recipe)

### Step 1 — Get an access token

1. Sign in to https://zenodo.org (or create an account)
2. Go to https://zenodo.org/account/settings/applications/tokens/new/
3. Create a token with `deposit:write` scope
4. Save as `$ZENODO_TOKEN` env var

### Step 2 — Create a new deposition

```bash
curl -H "Content-Type: application/json" \
     -H "Authorization: Bearer $ZENODO_TOKEN" \
     -X POST \
     -d '{
       "metadata": {
         "title": "SLC35G1 v10 final + v11 reproducibility package",
         "upload_type": "publication",
         "publication_type": "article",
         "publication_date": "2026-06-22",
         "description": "<see §2 above>",
         "creators": [{"name": "Chen, J.", "affiliation": "..."}],
         "keywords": ["precision oncology", "sialylation", "SLC35G1", "multi-omics"],
         "access_right": "open",
         "license": "cc-by-4.0",
         "version": "v1.0"
       }
     }' \
     https://zenodo.org/api/deposit/depositions
```

Save the `id` from the response as `$DEPOSITION_ID`.

### Step 3 — Upload files

```bash
# Upload the data bundle (9 GB; may take 30+ min)
curl -H "Authorization: Bearer $ZENODO_TOKEN" \
     -X POST \
     -F "file=@v11_data_bundle.zip" \
     https://zenodo.org/api/deposit/depositions/$DEPOSITION_ID/files

# Upload the paper
curl -H "Authorization: Bearer $ZENODO_TOKEN" \
     -X POST \
     -F "file=@v10_final_paper.pdf" \
     https://zenodo.org/api/deposit/depositions/$DEPOSITION_ID/files
```

### Step 4 — Publish

```bash
curl -H "Authorization: Bearer $ZENODO_TOKEN" \
     -X POST \
     https://zenodo.org/api/deposit/depositions/$DEPOSITION_ID/actions/publish
```

Zenodo returns the DOI in the form `10.5281/zenodo.PLACEHOLDER`.

```python
# Pseudo-code (to be implemented by 老刀):
import zipfile
from pathlib import Path

with zipfile.ZipFile("v11_data_bundle.zip", "w", zipfile.ZIP_DEFLATED) as zf:
    for h5ad in Path("data/").glob("*.h5ad"):
        zf.write(h5ad, arcname=h5ad.name)
    for pdb in Path("data/pdb/").glob("*.pdb"):
        zf.write(pdb, arcname=f"pdb/{pdb.name}")
    # Add SHA-256 manifest
    manifest = "\n".join(
        f"{sha256(h5ad)}  {h5ad.name}"
        for h5ad in Path("data/").glob("*.h5ad")
    )
    zf.writestr("MANIFEST.sha256", manifest)
```

## 7. Why Zenodo (vs Figshare / Dryad)?

| Aspect | Zenodo | Figshare | Dryad |
|---|---|---|---|
| **DOI minted automatically** | ✅ | ✅ | ✅ |
| **No curation fee** | ✅ free | ✅ free | ❌ $120+ |
| **Max file size per record** | 50 GB | 5 GB free tier | 300 GB |
| **GitHub integration** | ✅ webhook on release | ❌ | ❌ |
| **1-region standard** | ✅ widely cited | ⚠️ less common | ⚠️ biology-focused |
| **License variety** | ✅ all CC variants | ✅ all CC variants | ✅ all CC variants |

**Verdict**: Zenodo is the canonical 1-region reproducibility standard.
